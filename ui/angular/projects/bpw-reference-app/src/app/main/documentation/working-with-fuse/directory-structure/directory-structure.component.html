Directory Structure

WCM has a fairly simple directory structure. All source code that you will need stays inside the /src folder.

Once you build your app, a /dist folder will appear at the root which will contain the built app. You can simply upload the contents of it to your server to start running your app.

Source directory (/src)

The source folder has the general Angular CLI project structure along with an additional folder called /@wcm which contains the core elements of the WCM.

We recommend you not to touch /@wcm directory to easily update WCM in the future.

/app

This folder contains the AppComponent along with the following directories. Everything that being contained inside these folders are belong to your app and you can edit them however you like while building your app:

  • /fake-db: The fake database data files for WCM apps.
  • /ui-config: The main config file for configuring the WCM template.
  • /layout: Contains the template layout components.
  • /main: Example WCM apps and ready to use pages and page layouts.
  • /navigation: The main navigation data.
  • /store: Ngrx-Mail app related store files.
